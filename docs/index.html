Subnetter Documentation

Welcome to the Subnetter documentation. Subnetter is a powerful tool that automates hierarchical IPv4 CIDR allocation across cloud infrastructure, addressing one of the most challenging aspects of modern cloud architecture.

The IP Address Management Challenge

Managing IP address space in modern cloud environments presents several significant challenges that grow exponentially with scale:

IP Address Management Complexity

  • Manual allocation is error-prone: Hand-crafting CIDR blocks for hundreds of subnets across multiple regions leads to mistakes that can cause outages, security vulnerabilities, and costly remediation efforts.

  • Overlapping CIDRs create critical connectivity issues: Without a systematic approach, IP conflicts between networks prevent proper routing, break VPC peering, and cause unexpected application failures.

  • Multi-cloud environments multiply complexity: Organizations using AWS, Azure, and GCP simultaneously need a unified addressing strategy while respecting each provider’s unique networking constraints.

Scaling Challenges

  • Growing organizations quickly exhaust IP space: Without proper planning, organizations need painful re-IP projects as they expand to new regions or add more workloads.

  • Future-proofing is difficult: Planning for expansion across regions, accounts, and cloud providers requires complex CIDR math and foresight about future network requirements.

  • Consistent documentation is hard to maintain: Network diagrams and spreadsheets become outdated as infrastructure evolves, leading to confusion and errors during network changes.

Operational Overhead

  • Tracking allocated CIDRs across teams: Different teams deploy resources in different regions with no central coordination, leading to IP conflicts and inefficient utilization.

  • Limited visibility across environments: Network administrators struggle to understand the global IP addressing picture, especially in organizations with multiple accounts or cloud providers.

  • Automation integration requires structured data: Infrastructure-as-code tools need consistent, predictable IP allocation to properly automate network provisioning.

How Subnetter Solves These Problems

Subnetter addresses these challenges through an innovative, deterministic approach to IP address allocation:

1. Hierarchical Allocation Model

Subnetter implements a structured, hierarchical approach to CIDR allocation that mirrors the organizational structure of modern cloud deployments:

This hierarchical model ensures:

  • No CIDR overlaps: Each level of the hierarchy has its own dedicated address space
  • Efficient allocation: Subnet sizes are tailored to their specific use cases
  • Future-proofing: Space is reserved at each level for future expansion
  • Natural organization mapping: Network design follows organizational boundaries

2. Deterministic, Configuration-Driven Approach

Subnetter uses a deterministic algorithm to generate CIDR allocations based on a single configuration file. This approach:

  • Eliminates human error in CIDR calculations and assignments
  • Ensures reproducibility across environments and over time
  • Provides a single source of truth for network architecture
  • Simplifies infrastructure-as-code integration

3. Multi-Cloud Flexibility with Consistent Patterns

Subnetter supports multiple cloud providers while maintaining a consistent addressing scheme:

  • Provider-specific configurations account for unique requirements of AWS, Azure, and GCP
  • Consistent patterns across environments simplify operations and documentation
  • Simple integration with cloud-specific infrastructure-as-code tools

4. Documentation and Visibility

Subnetter automatically generates detailed allocation outputs that:

  • Document all CIDR allocations in a standardized format
  • Provide clear visibility into the entire network hierarchy
  • Support automation through machine-readable outputs
  • Prevent IP conflicts by making allocations explicit and traceable

Key Features

  • Hierarchical IPv4 CIDR allocation (Provider → Account → Region → AZ → Subnet)
  • Support for multiple cloud providers (AWS, Azure, GCP)
  • Deterministic allocation to prevent overlaps
  • Configurable prefix lengths for different hierarchy levels
  • Variable subnet sizes based on workload requirements
  • CSV output for easy integration with other tools
  • Support for cloud-specific configurations per account

Use Cases

Subnetter is designed to help organizations:

  • Plan and manage IP address space across cloud environments
  • Ensure consistent, non-overlapping CIDR blocks
  • Automate network design for infrastructure as code
  • Document network architecture in a structured format
  • Scale network architecture across multiple cloud providers
  • Maintain a clear IP addressing scheme as infrastructure grows
